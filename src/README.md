# Source Directory Structure

This directory contains the Bizmanage customizations pulled from your platform instance using the `bizmanage pull` command.

## Directory Structure

```
src/
├── objects/          # Tables/Views and their Actions
├── backend/          # Server-side Scripts  
├── reports/          # Custom SQL Reports
└── pages/            # Custom HTML Pages
```

## Important Notes

⚠️ **Do not commit these files to version control!**

The contents of these directories are:
- Generated by running `bizmanage pull`
- Specific to your Bizmanage instance
- Should be pulled fresh by each developer from their own instance

## Usage

1. **Pull data**: `bizmanage pull`
2. **Edit files** in this directory structure
3. **Push changes**: `bizmanage push` (when implemented)

## What Gets Generated

### `objects/`
```
objects/
├── customers/
│   ├── definition.json       # Table schema and settings
│   └── actions/
│       ├── validate-vat.js       # JavaScript action
│       └── validate-vat.meta.json # Action metadata
└── orders/
    ├── definition.json
    └── actions/
        ├── approve.js
        └── approve.meta.json
```

### `backend/`
```
backend/
├── nightly-sync.js           # Script code
├── nightly-sync.meta.json    # Script metadata (triggers, dependencies)
├── shared-utils.js
└── shared-utils.meta.json
```

### `reports/`
```
reports/
├── q3-revenue.sql            # SQL query
├── q3-revenue.meta.json      # Report metadata (parameters, permissions)
├── customer-analysis.sql
└── customer-analysis.meta.json
```

### `pages/`
```
pages/
├── dashboard.html            # HTML content
├── dashboard.meta.json       # Page metadata (navigation, permissions)
├── admin-panel.html
└── admin-panel.meta.json
```

## File Types

- **`.js` files**: JavaScript code for actions and backend scripts
- **`.sql` files**: SQL queries for reports  
- **`.html` files**: HTML content for custom pages
- **`.json` files**: Configuration and metadata
- **`definition.json`**: Table/view schemas and settings
- **`.meta.json`**: Metadata for actions, scripts, reports, and pages
